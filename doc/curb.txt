==============================================================================
1. Introduction                                                 *curb-intro*

Curb provides a floating prompt for replacing the current visual selection.

==============================================================================
2. Installation                                               *curb-install*

Install the plugin with your preferred plugin manager, then call:

>
    require("curb").setup()
<

==============================================================================
3. Setup                                                         *curb-setup*

Minimal setup:

>
    require("curb").setup()
<

With options:

>
    require("curb").setup({
      trigger_key = "<leader>ai",
      accept_key = "<C-y>",
      provider = {
        endpoint = "https://ai.hackclub.com/proxy/v1/chat/completions",
        model = "qwen/qwen3-coder-plus",
        api_key_env = "HACKCLUB_API_KEY",
        api_key_file = nil,
      },
      highlights = {
        normal = "Normal",
        border = "Keyword",
        title_icon = "DiagnosticInfo",
        title_text = "Keyword",
        footer = "Comment",
      },
    })
<

The setup function creates:

- The |:Curb| user command.
- A visual mode keymap that opens the prompt for the current selection.

==============================================================================
4. Usage                                                         *curb-usage*

Select text in visual mode and use one of the following:

- Run |:Curb|
- Press the configured `trigger_key`

Inside the floating prompt, press the configured `accept_key` to apply the
replacement.

                                                         *:Curb*
:Curb   Open the Curb prompt for the current visual selection.

                                                        *:CurbSetApiKey*
:CurbSetApiKey
        Prompt for an API key and save it to Curb's local key file.

                                                     *:CurbApiKeyStatus*
:CurbApiKeyStatus
        Report whether Curb can currently resolve an API key.

==============================================================================
5. Configuration                                               *curb-config*

                                                        *g:curb-config*
Available setup keys:

`trigger_key`                                              *curb-config-trigger*
    Type: string
    Default: >
        "<leader>ai"
<
    Visual mode mapping used to open the prompt.

`accept_key`                                                *curb-config-accept*
    Type: string
    Default: >
        "<C-y>"
<
    Insert mode mapping used inside the prompt window to apply the result.

`provider`
    Type: table
    Backend request settings used for the async `curl` call.

    Supported keys:

    - `endpoint`: Chat Completions API URL
    - `model`: model identifier sent in the request body
    - `api_key_env`: environment variable name used to read the API key
    - `api_key_file`: optional path to a file containing the API key

`highlights`                                             *curb-config-highlights*
    Type: table
    Highlight definitions for the floating window UI.

==============================================================================
6. Highlights                                             *curb-highlights*

Each highlight entry under `highlights` may be:

- An existing highlight group name
- A hex color string like `#89b4fa`
- A table of highlight attributes accepted by |nvim_set_hl()|

Supported keys:

- `normal`
- `border`
- `title_icon`
- `title_text`
- `footer`
